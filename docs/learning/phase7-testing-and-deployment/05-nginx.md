> 主题：Nginx 反向代理、连接管理、上传与安全边界。

# 背景

直接暴露 Uvicorn 会让应用进程同时承担公网入口、连接策略和请求限制。Nginx 可作为稳定入口处理反向代理、TLS、请求大小、超时和基础限流。

它不会自动修复应用权限或性能问题，但能把通用边缘职责从业务服务分离。

# 核心思想

反向代理代表客户端与上游应用通信。客户端只看到 Nginx，Nginx 根据规则选择上游并传递必要上下文。

信任边界很关键：应用只有在请求确实来自受信代理时，才能相信转发的客户端 IP、协议和主机头。

# 工作原理

Nginx 事件驱动 Worker 接收连接，解析请求并转发到 Uvicorn。它可缓冲请求/响应、复用上游连接、终止 TLS，并根据超时与大小规则拒绝异常流量。

`X-Forwarded-*` 等 Header 传递原始连接信息，但外部客户端也能伪造；入口必须覆盖或清理，再由应用配置受信代理。

# 企业实践

- TLS 在可信入口终止，HTTP 重定向和证书更新自动化。
- 设置请求体、上传、读取和连接超时。
- 只转发必要 Header，正确处理真实客户端 IP。
- 健康检查路径轻量，不泄露配置与依赖细节。
- 限流作为保护层，业务级配额仍由应用处理。
- 日志携带 Request ID，并避免记录 Token 和敏感查询参数。

# TaskFlow Pro应用

Nginx 暴露统一 API 入口并代理到 `api` 服务。文件上传设置与 TaskFlow 限额一致，AI 长任务通过异步接口避免超长代理等待。

Uvicorn 只信任 Compose 内的 Nginx 代理，生成绝对 URL 或安全 Cookie 时正确识别外部 HTTPS。

# 常见误区

- 信任任意来源的 `X-Forwarded-For`。
- Nginx 上传限制与应用限制不一致。
- 为 LLM 请求无限增加代理超时。
- 认为 Nginx 限流能替代用户权限和业务配额。
- 健康接口查询所有依赖并返回敏感错误。
